# Purpose
웹/모바일웹 기능을 사용자 시나리오 기준으로 점검한다.

# When to use
- 새 화면 개발 후
- 로그인, 목록, 상세, 저장, 수정, 삭제 흐름을 확인할 때
- 모바일웹 동작과 UI 전환을 점검할 때
- 주요 사용자 시나리오가 정상적으로 이어지는지 확인할 때

# Workflow
1. 현재 프로젝트에서 테스트 가능한 화면 또는 진입 경로를 파악한다.
2. 핵심 사용자 흐름을 3~5개 선정한다.
3. 정상 케이스, 예외 케이스, 경계값 케이스를 나누어 점검한다.
4. 각 단계의 기대 결과와 실제 결과를 비교한다.
5. 실패 시 재현 절차, 영향 범위, 원인 후보를 정리한다.
6. 필요하면 관련 코드 위치와 수정 포인트를 제안한다.

# Output format
- 테스트 대상
- 시나리오 목록
- 통과/실패 결과
- 발견 이슈
- 재현 절차
- 권장 조치
