# .impeccable.md — Hola PMS Design Context

## Design Context

### Users
- **Primary**: 호텔 운영 관리자 (프론트데스크, 객실관리, 예약팀). 하루 8시간+ PMS 앞에서 근무
- **Secondary**: 하우스키핑 직원 (모바일 앱), 호텔 관리자 (대시보드/리포트)
- **Context**: 바쁜 체크인/아웃 시간에 빠르고 정확한 조작 필요. 실수 비용이 높음 (오버부킹, 잘못된 요금 등)
- **Job to be done**: 객실 재고, 예약, 요금, 하우스키핑을 한 곳에서 실수 없이 관리
- **Tech proficiency**: 다양 — IT 전문가가 아닌 호텔 실무자. 새 직원 온보딩이 잦음

### Brand Personality
- **3 words**: 전문적 (Professional) · 깔끔 (Clean) · 실용적 (Practical)
- **Voice**: 간결하고 명확. 불필요한 장식 없이 정보를 전달. 업계 용어를 자연스럽게 사용
- **Emotional goal**: 신뢰감 + 안정감 — "이 시스템은 믿을 수 있다"는 확신
- **Anti-reference**: 화려하지만 느린 UI, 과도한 애니메이션, 불필요한 일러스트/마스코트

### Aesthetic Direction
- **Visual tone**: 절제된 SaaS 스타일. 기능이 디자인을 이끄는 접근
- **Reference**: Cloudbeds, Mews PMS — 업계 표준적 호텔 PMS 인터페이스
- **Theme**: Light mode (콘텐츠 영역), Dark sidebar/header (#051923, #003554)
- **Typography**: Pretendard (한글 최적화 sans-serif). 본문 0.875rem, 컴팩트하지만 가독성 확보
- **Color system**: 
  - Primary: `#0582CA` (신뢰의 파란색)
  - Dark: `#051923` → `#003554` (네이비 그라데이션)
  - Danger: `#EF476F` (주의가 필요한 액션)
  - Neutral: `#f8f9fa` ~ `#212529` (회색 스케일)
- **Shape**: 둥근 모서리 (6-14px radius), 부드러운 그림자. 날카롭지 않지만 과하게 둥글지도 않음

### Design Principles

1. **Clarity over decoration** — 모든 UI 요소는 명확한 목적을 가져야 한다. 장식적 요소보다 정보 전달이 우선. 라벨, 상태, 액션이 한눈에 구분되어야 한다
2. **Density with breathing room** — 호텔 운영에 필요한 정보 밀도를 유지하되, 적절한 여백으로 시각적 피로를 방지. 테이블, 폼, 카드 사이에 일관된 간격 체계 사용
3. **Consistent patterns, zero surprises** — 동일한 액션은 어디서든 동일하게 동작하고 동일하게 보여야 한다. 버튼 위치, 폼 레이아웃, 모달 구조, 배지 스타일이 시스템 전체에서 통일
4. **Progressive disclosure** — 핵심 정보를 먼저 보여주고, 상세 정보는 필요할 때 접근. 복잡한 워크플로를 단계별로 분리하여 인지 부하 최소화
5. **Fail-safe interactions** — 되돌릴 수 없는 액션에는 확인 단계를 둔다. 상태 변경은 시각적으로 명확히 피드백. 에러 메시지는 무엇이 잘못됐고 어떻게 고칠 수 있는지를 안내

### Accessibility
- WCAG AA level compliance
- Sufficient color contrast (4.5:1 text, 3:1 UI components)
- Keyboard navigable forms and modals
- Proper focus indicators on interactive elements
- No motion-dependent interactions (reduced motion safe)
