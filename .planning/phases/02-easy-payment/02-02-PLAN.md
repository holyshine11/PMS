In `booking.js` CheckoutPage, add a `pageshow` event handler in the `bindEvents` function (around line 754, after the existing `window.addEventListener('message', ...)` block).

**Purpose:** When mobile user returns from KICC auth page via `window.history.back()`, the page may be restored from bfcache. The `postMessage` handler won't fire because `billkey-return.html` uses `window.history.back()` for mobile (no opener). We need `pageshow` to detect this case, check sessionStorage for a pending billkey registration, poll the result endpoint, and refresh cards.

**Add this code block inside `bindEvents`, after the existing postMessage listener (after line 754):**

```javascript
// 모바일: KICC 빌키 등록 후 페이지 복귀 시 카드 목록 갱신
window.addEventListener('pageshow', function(e) {
    var shopOrderNo = sessionStorage.getItem('hola_billkey_shopOrderNo');
    if (!shopOrderNo) return;

    // bfcache 복원 또는 일반 네비게이션 모두 처리
    sessionStorage.removeItem('hola_billkey_shopOrderNo');

    // 빌키 등록 결과 폴링
    var pollCount = 0;
    var maxPoll = 10;
    var pollInterval = 1500; // 1.5초

    function pollBillkeyResult() {
        $.ajax({
            url: API_BASE + '/easy-pay/billkey-result',
            method: 'GET',
            data: { shopOrderNo: shopOrderNo }
        }).done(function(res) {
            if (res.status === 'SUCCESS') {
                self.loadEasyPayCards();
                showSuccess('카드가 등록되었습니다.');
            } else if (res.status === 'FAILED') {
                showError('카드 등록에 실패했습니다.');
            } else if (res.status === 'PENDING' && pollCount < maxPoll) {
                pollCount++;
                setTimeout(pollBillkeyResult, pollInterval);
            } else {
                // 타임아웃: 그래도 카드 목록 갱신 시도
                self.loadEasyPayCards();
            }
        }).fail(function() {
            // 폴링 실패 시에도 카드 목록 갱신 시도
            self.loadEasyPayCards();
        });
    }

    pollBillkeyResult();
});
```

**Important:** The `self` variable is already declared at the top of `bindEvents` as `var self = this;` (line 704). The `showSuccess` and `showError` functions should use the existing `showError` pattern already in the file (search for existing usage). If `showSuccess` doesn't exist, use `HolaPms.alert('success', '카드가 등록되었습니다.')` if HolaPms is available in booking context, otherwise just skip the success message and only call `self.loadEasyPayCards()`.

Also check if `showSuccess` is defined in booking.js — if not, replace `showSuccess('카드가 등록되었습니다.')` with just the `loadEasyPayCards()` call (the card appearing in the list is sufficient feedback).
