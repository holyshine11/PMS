# Hola PMS 대시보드 리디자인 — 디자인 원칙 및 기준

> **문서 목적**: 구현 전 디자인 원칙을 확정하여, 모든 수정 작업이 일관된 기준 하에 진행되도록 함
> **적용 범위**: 대시보드 (SUPER_ADMIN / HOTEL_ADMIN / PROPERTY_ADMIN)
> **철학**: Dieter Rams — "Less, but better"
> **작성일**: 2026-03-23

## 1. 컬러 시스템

### 1.1 기존 팔레트 (유지)

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--hola-sidebar-bg` | `#051923` | 가장 어두운 네이비. 사이드바 배경 |
| `--hola-header-bg` | `#003554` | 어두운 블루. 헤더 배경, 보조 강조 |
| `--hola-accent` | `#0582CA` | 밝은 블루. **Primary 액션 & 긍정 지표** |
| `--hola-danger` | `#EF476F` | 핑크 레드. **주의/경고/부정 지표** |
| Neutral | `#000` / `#FFF` + gray 계열 | 텍스트, 배경 |

### 1.2 대시보드 컬러 원칙

#### 규칙 1: 의미 기반 컬러만 사용 (장식적 컬러 금지)

```
✅ 허용:
- #0582CA (accent)  → 긍정적 수치, 상승, 목표 달성
- #EF476F (danger)  → 부정적 수치, 하락, 주의 필요
- #495057 (gray-700) → 중립적 수치, 비교 기준
- #212529 (gray-900) → 기본 텍스트

❌ 금지:
- 카드마다 다른 배경색 (무지개 효과)
- bg-success / bg-info / bg-warning 등 Bootstrap 컨텍스트 컬러를 장식 목적으로 사용
- 의미 없는 컬러 코딩 (예: 도착=파랑, 투숙=네이비, 출발=핑크)
```

#### 규칙 2: 60-30-10 비율

```
60%  — 배경 & 중립 (#FFF, #f8f9fa, #f1f3f5)
30%  — 텍스트 & 구조 (#212529, #495057, #6c757d, 보더)
10%  — 액센트 (#0582CA) + 위험 (#EF476F)
```

#### 규칙 3: 트렌드 표시 컬러 (신규 정의)

| 상태 | 컬러 | CSS 변수 | 용도 |
|------|------|----------|------|
| 상승/긍정 | `#0582CA` | `--hola-accent` | ▲ 전일 대비 상승 |
| 하락/부정 | `#EF476F` | `--hola-danger` | ▼ 전일 대비 하락 |
| 변동 없음 | `#6c757d` | `--hola-text-secondary` | — 변동 없음 |

## 2. 타이포그래피 시스템

### 2.1 대시보드 타이포 스케일

| Level | 용도 | Size | Weight | Color |
|-------|------|------|--------|-------|
| **Hero Metric** | OCC% (히어로 KPI) | `2rem` (32px) | 700 | `#212529` |
| **KPI Value** | ADR, RevPAR, 매출 등 | `1.5rem` (24px) | 700 | `#212529` |
| **KPI Label** | 지표 이름 | `0.75rem` (12px) | 500 | `#6c757d` |
| **KPI Sub** | 보조 정보 (x/y 객실) | `0.8125rem` (13px) | 400 | `#6c757d` |
| **Trend** | 전일 대비 변화 | `0.75rem` (12px) | 600 | accent/danger |
| **Section Title** | 영역 제목 | `0.875rem` (14px) | 600 | `#212529` |
| **Table Header** | 테이블 열 제목 | `0.75rem` (12px) | 600 | `#495057` |
| **Table Cell** | 테이블 데이터 | `0.8125rem` (13px) | 400 | `#212529` |
| **Date Display** | 영업일 표시 | `0.875rem` (14px) | 600 | `#003554` |

### 2.2 타이포 원칙

```
✅ 허용:
- 한글 라벨은 text-transform: none (uppercase 금지)
- KPI 라벨은 간결한 한글 ("점유율", "객단가", "매출")
- 영문 약어는 라벨 뒤 괄호 ("점유율 (OCC)")

❌ 금지:
- text-transform: uppercase를 한글에 적용
- KPI 값과 라벨의 크기 차이 1:2 이상 (현재 1:1.83 → 위계 점프 과도)
- 모든 KPI 값에 동일한 font-size (히어로 메트릭 구분 필요)
```

## 3. 스페이싱 시스템

### 3.1 기본 단위: 8px 그리드

| Token | Size | 용도 |
|-------|------|------|
| `xs` | 4px | 아이콘-텍스트 간격 |
| `sm` | 8px | 카드 내부 요소 간격, 관련 항목 사이 |
| `md` | 16px | 카드 내부 패딩, 형제 카드 간격 |
| `lg` | 24px | **섹션 간 간격** (KPI → 운영현황 → 차트) |
| `xl` | 32px | 대시보드 상단/하단 여백 |
| `2xl` | 48px | 역할 전환 시 영역 분리 |

### 3.2 스페이싱 원칙 — Proximity (근접성)

```
✅ 적용:
- 같은 카테고리 카드: gap 16px (g-3)
- 다른 카테고리 영역: gap 32px (mb-5) + 섹션 헤더
- 섹션 헤더와 콘텐츠: gap 12px

❌ 금지:
- 모든 영역에 동일한 mb-4 (24px) — 구분 없는 나열
- 의미 다른 영역을 간격만으로 구분 (섹션 헤더 병행 필요)
```

### 3.3 섹션 구분 패턴

```html
<!-- 섹션 구분: 아이콘 + 소제목 + 충분한 상하 여백 -->
<div class="mb-5">  <!-- 영역 간 32px -->
    <h6 class="fw-bold text-secondary mb-3">
        <i class="fas fa-chart-bar me-2"></i>핵심 지표
    </h6>
    <!-- 카드들... -->
</div>

<div class="mb-5">
    <h6 class="fw-bold text-secondary mb-3">
        <i class="fas fa-clipboard-check me-2"></i>운영 현황
    </h6>
    <!-- 카드들... -->
</div>
```

## 4. 카드 컴포넌트 시스템

### 4.1 단일 카드 패턴 (통일)

**모든 대시보드 카드는 하나의 스타일만 사용**:

```css
.dashboard-card {
    border: none;
    border-radius: var(--hola-radius-md);    /* 10px */
    box-shadow: var(--hola-shadow-sm);
    background: #fff;
    transition: transform 0.15s ease, box-shadow 0.15s ease;
}
.dashboard-card:hover {
    transform: translateY(-2px);
    box-shadow: var(--hola-shadow-md);
}
```

### 4.2 카드 변형 (용도별)

| 변형 | 시각 구분 | 용도 |
|------|-----------|------|
| **기본형** | 흰 배경 + shadow-sm | KPI, 운영현황 |
| **히어로형** | 기본형 + 좌측 4px 액센트 보더 | 최상위 KPI (OCC) |
| **액션형** | 기본형 + cursor:pointer + 호버 강조 | 클릭 가능 카드 (운영현황) |
| **랭킹형** | 카드 내부 테이블 | SUPER_ADMIN 랭킹 |

### 4.3 카드 원칙

```
✅ 허용:
- 모든 카드에 동일한 border-radius, shadow, padding
- 히어로 카드만 좌측 액센트 보더로 구분
- 아이콘은 카드 우측 상단에 작고 은은하게 (또는 생략)

❌ 금지:
- 카드마다 다른 아이콘 배경색 (무지개 금지)
- 카드마다 다른 좌측 보더 색 (3색 이상)
- inline style로 보더/색상 정의
- 원형 아이콘 배경 (bg-xxx bg-opacity-10 패턴)
```

## 5. 시각적 위계 (Visual Hierarchy)

### 5.1 대시보드 정보 위계

```
Level 1 (Focal Point)  → 영업일 + 히어로 메트릭 (OCC%)
Level 2 (Primary)      → 보조 KPI (ADR, RevPAR, 매출)
Level 3 (Secondary)    → 운영현황 (도착, 투숙, 출발)
Level 4 (Context)      → 추이 차트, 랭킹 테이블
Level 5 (Meta)         → 빠른 메뉴, 툴바
```

### 5.2 위계 표현 수단

| Level | 크기 | 위치 | 강조 |
|-------|------|------|------|
| L1 | 큰 폰트 (2rem) | 최상단 | 액센트 보더 |
| L2 | 중간 폰트 (1.5rem) | L1 하단 | 기본 카드 |
| L3 | 기본 폰트 (1.25rem) | L2 하단 | 섹션 헤더 구분 |
| L4 | 차트/테이블 | L3 하단 | 카드 내 콘텐츠 |
| L5 | 작은 폰트 (0.8rem) | 맥락에 따라 | 보조 UI |

### 5.3 읽기 흐름 (F-Pattern)

```
┌─────────────────────────────────────────────────────┐
│  [영업일: 2026년 3월 23일 (월)]                      │ ← L1: 컨텍스트 설정
├─────────────────────────────────────────────────────┤
│  [■ OCC 히어로 카드 — 전체 너비 or 2/3]              │ ← L1: 최초 시선 집중
├─────────────────────────────────────────────────────┤
│  [ADR 카드] [RevPAR 카드] [매출 카드]                │ ← L2: 왼→오 스캔
├─────────────────────────────────────────────────────┤
│  ── 운영 현황 ──                                     │
│  [도착 예정] [현재 투숙] [출발 예정]                  │ ← L3: 운영 확인
├─────────────────────────────────────────────────────┤
│  ── 예약 추이 ──                                     │
│  [7일 추이 차트 — 전체 너비]                         │ ← L4: 트렌드 파악
└─────────────────────────────────────────────────────┘
```

## 6. 데이터 표현 원칙

### 6.1 맥락 없는 숫자 금지 (Tufte's Principle)

모든 KPI 수치에는 **비교 기준**이 반드시 포함되어야 함:

```
✅ 좋은 예:
┌──────────────────────┐
│  점유율 (OCC)         │
│  62.3%        ▲ 2.1% │
│  280/450실    vs 어제 │
└──────────────────────┘

❌ 나쁜 예:
┌──────────────────────┐
│  점유율 (OCC%)        │
│  62.3%                │
│                       │
└──────────────────────┘
```

### 6.2 비교 기준 우선순위

| 순위 | 비교 기준 | 적용 KPI |
|------|-----------|----------|
| 1순위 | 전일 대비 (vs 어제) | OCC, ADR, RevPAR, 매출 |
| 2순위 | 분모/분자 표시 | OCC (280/450실), 도착(체크인 5/15) |
| 3순위 | 프로그레스 바 | OCC, 작업 완료율 |

### 6.3 트렌드 표시 패턴

```html
<!-- 상승 -->
<span class="trend-up">
    <i class="fas fa-arrow-up"></i> 2.1%
</span>

<!-- 하락 -->
<span class="trend-down">
    <i class="fas fa-arrow-down"></i> 3.5%
</span>

<!-- 변동 없음 -->
<span class="trend-flat">
    <i class="fas fa-minus"></i> 0.0%
</span>
```

```css
.trend-up   { color: #0582CA; font-size: 0.75rem; font-weight: 600; }
.trend-down { color: #EF476F; font-size: 0.75rem; font-weight: 600; }
.trend-flat { color: #6c757d; font-size: 0.75rem; font-weight: 500; }
```

### 6.4 프로그레스 바 규칙 (OCC 시각화)

```css
.occ-progress {
    height: 6px;
    border-radius: 3px;
    background-color: #e9ecef;
}
.occ-progress-bar {
    border-radius: 3px;
    background-color: #0582CA;
    transition: width 0.6s ease;
}
/* 80% 이상: 긍정 강조 없음 — 기본 accent 유지 */
/* 30% 미만: danger 컬러로 전환 */
.occ-progress-bar.low { background-color: #EF476F; }
```

## 7. 빈 상태 (Empty State) 디자인

### 7.1 원칙: 빈 화면은 기회

```
✅ 빈 상태가 제공해야 하는 것:
1. 인사/환영 → 사용자 이름
2. 안내 → 다음 행동 (프로퍼티 선택)
3. 바로가기 → 최근 사용 프로퍼티
4. 빠른 메뉴 → 자주 쓰는 기능

❌ 빈 상태에서 해서는 안 되는 것:
- 단순 alert 메시지 1줄만 표시
- 빈 공간 방치
- "데이터가 없습니다" 류의 소극적 안내
```

### 7.2 빈 상태 레이아웃

```
┌──────────────────────────────────────────────────────┐
│                                                       │
│   안녕하세요, {이름}님 👋                             │
│   ────────────────────                                │
│   헤더에서 호텔과 프로퍼티를 선택하면                  │
│   대시보드가 활성화됩니다.                             │
│                                                       │
│   ┌─ 최근 사용 ──────────────────────────────────┐   │
│   │  🏨 그랜드 호텔 서울 - 본관     [바로가기 →]  │   │
│   │  🏨 그랜드 호텔 서울 - 별관     [바로가기 →]  │   │
│   └──────────────────────────────────────────────┘   │
│                                                       │
│   ┌─ 빠른 메뉴 ─────────────────────────────────┐   │
│   │  📋 운영현황    🛏 객실현황    📅 예약 관리   │   │
│   └──────────────────────────────────────────────┘   │
│                                                       │
└──────────────────────────────────────────────────────┘
```

## 8. 인터랙션 패턴

### 8.1 클릭 가능 영역 (Affordance)

```
✅ 클릭 가능한 요소에는 반드시:
- cursor: pointer
- hover 시 시각적 변화 (shadow 강화 또는 배경색 변화)
- 화살표 아이콘(→) 또는 시각적 힌트

❌ 금지:
- 클릭 가능한데 hover 효과 없음
- 전체 카드가 클릭 가능한데 특정 텍스트만 링크 스타일
```

### 8.2 운영현황 카드 → 액션 연결

| 카드 | 클릭 시 이동 |
|------|-------------|
| 도착 예정 | `/admin/front-desk/operations` (도착 필터) |
| 현재 투숙 | `/admin/front-desk/operations` (투숙 필터) |
| 출발 예정 | `/admin/front-desk/operations` (출발 필터) |

### 8.3 SUPER_ADMIN 랭킹 → 드릴다운

| 액션 | 동작 |
|------|------|
| 프로퍼티명 클릭 | 해당 프로퍼티 자동 선택 → 프로퍼티 대시보드 표시 |
| 행 hover | 배경 하이라이트 + cursor:pointer |

## 10. 차트 디자인 원칙

### 10.1 Chart.js 스타일 가이드

```javascript
// 컬러: Hola 팔레트만 사용
const CHART_COLORS = {
    bar: 'rgba(5, 130, 202, 0.65)',      // accent 65% 투명
    barBorder: '#0582CA',
    line: '#EF476F',
    lineFill: 'rgba(239, 71, 111, 0.08)', // danger 8% 투명
    grid: '#f1f3f5',
    text: '#6c757d'
};

// 폰트: 전역 Pretendard
Chart.defaults.font.family = 'Pretendard';
Chart.defaults.font.size = 12;
Chart.defaults.color = '#6c757d';
```

### 10.2 차트 원칙

```
✅ 허용:
- 최대 2개 데이터셋 (복잡도 제한)
- y축 라벨에 단위 포함 ("객실 수", "매출(만원)")
- 호버 툴팁에 상세 수치

❌ 금지:
- 3D 효과, 그라디언트 과다 사용
- 범례(legend)가 차트보다 큰 경우
- height 하드코딩 (responsive 사용)
```

## 11. 반응형 원칙

### 11.1 브레이크포인트

| 화면 | 처리 |
|------|------|
| `≥1200px` (xl) | KPI 4열, 운영현황 3열 |
| `992-1199px` (lg) | KPI 2열 → 2행, 운영현황 3열 |
| `768-991px` (md) | KPI 2열, 운영현황 1열 |
| `<768px` (sm) | 전부 1열 (모바일 접근 시) |

### 11.2 차트 반응형

```
✅ 규칙:
- height 속성 제거, CSS로 min-height만 지정
- 모바일에서는 y1 (매출) 축 숨김 → 단일 축
- 범례는 하단 배치 (좁은 화면에서 상단 범례는 차트 영역 잠식)
```

## 12. 접근성 원칙

### 12.1 컬러 접근성

```
✅ 규칙:
- 트렌드 표시: 컬러 + 아이콘(▲/▼/—) 병행 (색맹 대응)
- OCC 뱃지: 컬러 + 텍스트 라벨 병행 ("높음", "보통", "낮음")
- 대비 비율: 텍스트 최소 4.5:1, 대형 텍스트 3:1

❌ 금지:
- 컬러만으로 상태 구분 (반드시 아이콘 또는 텍스트 병행)
```

### 12.2 키보드 접근성

```
✅ 규칙:
- 클릭 가능 카드: tabindex="0" + Enter/Space 키 대응
- focus-visible 스타일: outline 2px solid #0582CA
- 차트: aria-label로 데이터 요약 제공
```

## 13. 구현 시 체크리스트

구현 단계에서 매 카드/섹션마다 아래 체크리스트를 확인:

- [ ] **위계**: 이 요소의 Level(1~5)은 명확한가?
- [ ] **컬러**: 의미 기반 컬러만 사용했는가? 장식 컬러는 없는가?
- [ ] **타이포**: 정의된 스케일 내에서 선택했는가?
- [ ] **스페이싱**: 같은 카테고리 요소끼리 근접, 다른 카테고리는 분리되어 있는가?
- [ ] **맥락**: 모든 수치에 비교 기준이 있는가?
- [ ] **일관성**: 같은 역할의 요소가 같은 스타일인가?
- [ ] **액션**: 클릭 가능한 요소에 affordance가 있는가?
- [ ] **빈 상태**: 데이터 없을 때 의미 있는 대안을 제공하는가?
- [ ] **접근성**: 컬러 외 아이콘/텍스트 대안이 있는가?

## 14. Anti-Patterns (절대 금지)

| # | Anti-Pattern | 이유 | 대안 |
|---|-------------|------|------|
| 1 | 카드별 다른 아이콘 배경색 | 무지개 효과 → 촌스러움 | 모노크롬 or 아이콘 생략 |
| 2 | `bg-xxx bg-opacity-10` 원형 아이콘 | 2018년 트렌드, 현재 구식 | 심플 아이콘 or 생략 |
| 3 | 한글에 `text-transform: uppercase` | 한글에 무의미, 가독성 저하 | `text-transform: none` |
| 4 | `style="border-left: 4px solid #xxx !important"` | 유지보수 불가, 인라인 스타일 | CSS 클래스 분리 |
| 5 | 모든 영역 동일 `mb-4` | 위계 구분 불가 | 섹션별 다른 간격 |
| 6 | 빈 화면에 alert 1줄 | 가치 전달 실패 | 웰컴 상태 디자인 |
| 7 | KPI에 비교 기준 없음 | 판단 불가 숫자 나열 | 전일 대비 △ 포함 |
| 8 | Bootstrap 기본 table 스타일 그대로 사용 | 평범한 느낌 | 프로그레스 바, 시각화 가미 |
