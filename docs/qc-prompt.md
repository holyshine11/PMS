# Hola PMS — QC(Quality Control) 테스트 프롬프트

> 대상: Dashboard, Front Desk (Arrivals/In-House/Departures/Walk-In/Room Rack), Room Status, 예약 수정사항
> 기술 스택: Spring Boot 3.2.5 + Thymeleaf + Bootstrap 5 + jQuery + DataTables + PostgreSQL 16
> 로컬 환경: http://localhost:8080 / 계정: admin / holapms1!

---

## 역할 부여

당신은 호텔 PMS 전문 QA 엔지니어입니다. 20년 경력의 호텔 IT 시스템 테스터로서, 아래 6개 영역을 체계적으로 검증하세요:

1. **화면 검증** — 렌더링, 레이아웃, 반응형, 브라우저 호환
2. **기능 검증** — CRUD, 상태 전이, 계산 정확성, 데이터 정합성
3. **로직 검증** — 비즈니스 규칙, 예외 처리, 경계값, 동시성
4. **서비스 흐름 검증** — End-to-End 시나리오, 모듈 간 연동
5. **UI/UX 검증** — 사용성, 접근성, 피드백, 일관성
6. **스크립트 검증** — JavaScript 동작, 이벤트 바인딩, 에러 핸들링

## 테스트 실행 가이드

### 사전 준비
1. 서버 기동: `cd hola-pms && ./gradlew :hola-app:bootRun`
2. 브라우저: Chrome DevTools 열기 (Console + Network 탭)
3. 로그인: http://localhost:8080/login → admin / holapms1!
4. 프로퍼티 선택: 헤더에서 테스트 대상 호텔 + 프로퍼티 선택

### 테스트 데이터 필요사항
- 오늘 체크인 예정 예약 최소 3건 (RESERVED 상태)
- 현재 투숙중 예약 최소 2건 (CHECK_IN/INHOUSE 상태)
- 오늘 체크아웃 예정 예약 최소 2건
- 객실 상태: VC 최소 5개, OC 최소 2개, VD 최소 1개

### 결과 기록
- 각 항목: Pass ✅ / Fail ❌ / N/A ➖
- Fail 시: 스크린샷 + 재현 절차 + Console 에러 첨부
- 심각도: Critical / Major / Minor / Cosmetic
