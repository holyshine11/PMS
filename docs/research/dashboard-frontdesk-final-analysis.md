## 1. 역할별 대시보드 구성

### 1.1 역할 매핑 (확정)

```
Opera Cloud                    Hola PMS                    대시보드 레벨
─────────────────────────────────────────────────────────────────────────
Master Admin         →   SUPER_ADMIN (블루웨이브 본사)   Level 0: 포트폴리오 전체
Chain Admin          →   HOTEL_ADMIN                     Level 1: 호텔(체인) 단위
Hub Admin            →   HOTEL_ADMIN                     Level 1: (동일)
Property Admin       →   PROPERTY_ADMIN                  Level 2: 프로퍼티 단위
Front Desk Agent     →   향후 FRONT_DESK 역할 추가       Level 3: 오늘의 운영
```

### 1.2 역할별 대시보드 화면 구성

#### SUPER_ADMIN (블루웨이브 본사) — 포트폴리오 총괄 뷰

| 위젯 | 내용 |
|------|------|
| **프로퍼티 비교 랭킹 테이블** | 전체 프로퍼티의 OCC%(점유율) / ADR(평균객단가) / RevPAR(객실당수익) 비교 |
| **전체 객실 현황 요약** | 총 객실 수, 판매 가능 객실, 사용불가(OOO) 객실 합산 |
| **7일 예약 추이(Pickup) 차트** | 전체 프로퍼티 합산 예약 증감 추이 |
| **당일 예약/취소 건수** | 오늘 들어온 신규 예약 수, 취소 수 |

#### HOTEL_ADMIN (호텔 관리자) — 호텔 산하 프로퍼티 비교 뷰

| 위젯 | 내용 |
|------|------|
| **산하 프로퍼티 비교표** | 선택한 호텔 소속 프로퍼티들의 OCC%(점유율) / ADR(평균객단가) / RevPAR(객실당수익) |
| **객실 현황 요약** | 호텔 산하 전체 객실 현황 합산 |
| **7일 예약 추이(Pickup) 차트** | 호텔 단위 예약 증감 추이 |
| **당일 예약/취소 건수** | 호텔 단위 합산 |

#### PROPERTY_ADMIN (프로퍼티 관리자) — 단일 프로퍼티 운영 뷰

| 위젯 | 내용 |
|------|------|
| **당일 도착(Arrivals)** | 오늘 체크인 예정 예약 건수 |
| **당일 출발(Departures)** | 오늘 체크아웃 예정 예약 건수 |
| **투숙중(In-House)** | 현재 투숙 중인 객실/게스트 수 |
| **OCC%(점유율)** | 판매객실 ÷ 판매가능객실 × 100 |
| **ADR(평균객단가)** | 총 객실매출 ÷ 판매객실 수 |
| **RevPAR(객실당수익)** | 총 객실매출 ÷ 판매가능객실 = OCC% × ADR |
| **객실 상태 현황** | Clean(청소완료) / Dirty(청소필요) / OOO(사용불가) / OOS(일시중단) 별 객실 수 |
| **객실 요약** | 전체 객실 / 판매 가능 / OOO / OOS |
| **예약 활동** | 도착/투숙/출발 세부 현황 + 평균 체크인 시간 |
| **7일 예약 추이(Pickup) 차트** | 프로퍼티 예약 증감 추이 |

## 2. 호텔 업계 주요 용어 정리

### 2.1 핵심 KPI 지표

| 영문 | 한글 | 산출 공식 | 쉬운 설명 |
|------|------|-----------|-----------|
| **OCC%** | 점유율 (Occupancy Rate) | 판매객실 ÷ 판매가능객실 × 100 | 전체 방 중 몇 %가 팔렸는지 |
| **ADR** | 평균객단가 (Average Daily Rate) | 총 객실매출 ÷ 판매객실 수 | 팔린 방 한 개당 평균 얼마 받았는지 |
| **RevPAR** | 객실당수익 (Revenue Per Available Room) | 총 객실매출 ÷ 판매가능객실 | 모든 방(빈 방 포함)이 평균 얼마를 벌었는지 |

> **예시**: 100개 방 호텔, 80개 판매, 총 매출 1,600만원
> - OCC% = 80/100 = **80%**
> - ADR = 1,600만/80 = **20만원**
> - RevPAR = 1,600만/100 = **16만원** (= 80% × 20만원)

### 2.2 대시보드 타일 용어

| 영문 | 한글 | 설명 |
|------|------|------|
| **Arrivals** | 도착 (체크인 예정) | 오늘 도착 예정인 예약 건수 |
| **Departures** | 출발 (체크아웃 예정) | 오늘 출발 예정인 예약 건수 |
| **In-House** | 투숙중 | 현재 호텔에 머물고 있는 게스트/객실 수 |
| **Room Status** | 객실 상태 현황 | 청소완료/청소필요/사용불가 등 상태별 객실 수 |
| **Room Summary** | 객실 요약 | 전체 객실 수, 판매 가능, 사용불가, 일시중단 객실 수 |
| **Reservation Activity** | 예약 활동 | 도착/투숙/출발 예약의 세부 현황 (평균 체크인 시간 포함) |
| **Pickup** | 예약 증감 추이 | 특정 기간 대비 새로 들어온 예약 건수의 변화량 |
| **Walk-In** | 현장 투숙 | 사전 예약 없이 호텔에 와서 바로 투숙하는 경우 |
| **Day-Use** | 당일 사용 | 당일 체크인-체크아웃 (숙박 없이 낮 시간만 사용) |
| **No-Show** | 미도착 | 예약했지만 오지 않은 경우 |

## 3. Room Rack (객실현황) 상태 용어 정리

### 3.1 객실 상태 6종 — 한글 설명

| 영문 | 코드 | 한글 | 쉬운 설명 | 색상 |
|------|------|------|-----------|------|
| **Vacant Clean** | VC | 빈 방 - 청소완료 | 비어있고 청소도 끝남 → **바로 손님 받을 수 있음** | 🟢 초록 |
| **Vacant Dirty** | VD | 빈 방 - 청소필요 | 손님이 나갔는데 아직 청소 안 함 → **청소 후 판매 가능** | 🔴 빨강 |
| **Occupied Clean** | OC | 사용중 - 청소완료 | 손님이 머물고 있고 오늘 청소 끝남 | 🔵 파랑 |
| **Occupied Dirty** | OD | 사용중 - 청소필요 | 손님이 머물고 있고 오늘 청소 아직 안 함 | 🟣 보라 |
| **Out of Order** | OOO | 사용불가 | 수리/공사 등으로 아예 못 씀 → **재고에서 완전 제외** (점유율 계산에도 빠짐) | ⬛ 회색+빗금 |
| **Out of Service** | OOS | 일시중단 | 경미한 문제로 잠시 안 팔지만, 급하면 팔 수 있음 → **재고에는 포함** | 🔲 연회색 |

### 3.2 상태 조합 이해

객실 상태는 두 가지가 조합됩니다:

```
[누가 있는지]           ×    [청소 상태]
──────────────              ──────────────
Vacant (비어있음)            Clean (청소완료)
Occupied (손님 있음)         Dirty (청소필요)
                            Inspected (검사완료) ← Phase D에서 추가
                            Pickup (간단정리) ← Phase D에서 추가
```

### 3.3 상태 전이 흐름 (일상적인 순서)

```
[손님 체크아웃]
  → 사용중·청소완료(OC) → 빈 방·청소필요(VD)
     ↓
  [하우스키핑 청소]
     ↓
  빈 방·청소완료(VC) ← 이제 새 손님 받을 수 있음!
     ↓
  [새 손님 체크인]
     ↓
  사용중·청소완료(OC)
     ↓
  [하루 지남 - 야간감사]
     ↓
  사용중·청소필요(OD) ← 매일 밤 자동 전환
     ↓
  [하우스키핑 청소]
     ↓
  사용중·청소완료(OC)
```

## 4. 체크인 워크플로우 분석

### 4.1 Opera 15단계가 필요한 이유

Opera는 **전 세계 40,000+ 호텔**을 지원하는 시스템이라, 모든 호텔 유형과 법규를 커버해야 합니다:

| 단계 | Opera 기능 | 왜 필요한지 | Hola에서 필요한지 |
|------|-----------|-------------|:-:|
| 1 | 도착 예약 접근 | 예약 찾기 | **O** |
| 2 | 예약 검색/선택 | 게스트 특정 | **O** (→ Step 1로 통합) |
| 3 | 게스트 비즈니스카드 확인 | 회사 계정 연결 | X (국내 불필요) |
| 4 | 객실 배정 | 방 지정 | **O** |
| 5 | 예약 개요 검토 | 요금/숙박 확인 | **O** |
| 6 | 패키지/아이템 확인 | 부대서비스 패키지 | △ (향후) |
| 7 | 개인정보 보호 확인 | GDPR (유럽 법규) | X (국내 별도 처리) |
| 8 | 신분증 확인 | 여권/ID 스캔 | △ (향후 추가 가능) |
| 9 | 멤버십 확인 | 로열티 프로그램 | X (Phase 3) |
| 10 | 결제/수금 | 카드 승인, 선수금 | **O** |
| 10-1 | 사전결제 규칙 | 전액 선결제 정책 | △ (정책에 따라) |
| 11 | 동반 게스트 검증 | 아동 투숙 법규 | X (국내 불필요) |
| 12 | 등록카드 생성 | 종이/전자 서명 | △ (향후) |
| 13 | 바우처 생성 | 할인 바우처 발급 | X |
| 14 | 체크인 완료 | 최종 확인 | **O** |
| 15 | 선불카드 발급 | 호텔 선불카드 | X |

> **결론**: Opera 15단계 중 **필수 5~6개**, 나머지는 유럽 법규/대형 체인 전용 기능.
> Hola 7단계로 축소해도 **국내 3~5성급 호텔 운영에 전혀 문제 없음**.

### 4.2 Hola PMS 체크인 7단계 (확정안)

```
Step 1. 예약 선택
        - 프론트데스크 > 도착(Arrivals) 화면에서 오늘 체크인 예정 목록 조회
        - 게스트명/예약번호로 검색 → 선택

Step 2. 게스트 정보 확인
        - 예약자명, 연락처, 특별요청 표시
        - 수정 필요 시 인라인 편집

Step 3. 객실 배정
        - 기존 M07의 room-assign/availability API 재사용
        - 자동 추천 (VC 상태 우선) + 수동 선택
        - ★ 신규: 객실 하우스키핑 상태(VC/VD) 표시 → VD 객실은 배정 경고

Step 4. 요금 확인
        - 레이트코드, 숙박일수, 일별 요금, 총액 표시
        - 얼리체크인 수수료 자동 계산 표시 (기존 EarlyLateCheckService 활용)

Step 5. 결제 확인
        - 기존 M07의 결제 API 재사용
        - 예치금 현황 + 기결제액 + 잔액 표시
        - ★ 정책 옵션: "미결제 시 체크인 차단" ON/OFF (프로퍼티 설정)

Step 6. 메모 확인
        - 기존 M07의 메모(Memo) API 재사용
        - 예약 메모 목록 표시 + 신규 추가 가능

Step 7. 체크인 완료
        - 기존 M07의 PUT /{id}/status API 호출
        - 상태 전이: RESERVED → CHECK_IN → INHOUSE
        - ★ 신규: 객실 상태 자동 전환 (fo_status → OCCUPIED)
```

### 4.3 기존 예약관리(M07)와의 충돌/중복 분석

#### 4.3.1 중복되는 부분 (재사용)

| 기능 | 기존 M07 구현 | 프론트데스크에서 | 처리 방안 |
|------|-------------|-----------------|-----------|
| 상태 변경 API | `PUT /{id}/status` | 동일 API 호출 | **API 재사용** |
| 객실 배정 API | `GET /room-assign/availability` | 동일 API 호출 | **API 재사용** |
| 결제 API | `POST /payment/transactions` | 동일 API 호출 | **API 재사용** |
| 메모 API | `GET/POST /memos` | 동일 API 호출 | **API 재사용** |
| 얼리/레이트 요금 | `EarlyLateCheckService` | 자동 호출됨 | **로직 재사용** |
| 예약 상세 조회 | `GET /{id}` | 동일 API 호출 | **API 재사용** |

#### 4.3.2 충돌하는 부분 (수정 필요)

| 영역 | 현재 M07 | 프론트데스크 필요사항 | 수정 방안 |
|------|---------|---------------------|-----------|
| **체크인 전제조건** | 전제조건 없이 상태만 전환 | 객실 배정 필수 검증 | `changeStatus` 메서드에 체크인 시 `roomNumberId != null` 검증 추가 |
| **객실 상태 연동** | 하우스키핑 상태 무관 | VC(빈방·청소완료) 객실만 배정 가능 | 객실 배정 시 `hk_status = CLEAN` 조건 추가 |
| **체크아웃 정산** | 미결제도 체크아웃 가능 | 잔액 0 확인 필요 (선택적) | `changeStatus`에 정산 확인 옵션 추가 |
| **객실 상태 자동 전환** | 없음 | 체크인→OC, 체크아웃→VD | `changeStatus` 내 객실 상태 업데이트 로직 추가 |

#### 4.3.3 상충하지 않는 부분 (독립 구현)

| 기능 | 설명 |
|------|------|
| **프론트데스크 > 도착 화면** | 기존 예약 리스트와 다른 화면. "오늘 체크인 예정"만 필터링한 전용 뷰 |
| **프론트데스크 > 투숙 화면** | 기존 예약 리스트에 없는 기능. "현재 투숙중"만 필터링한 전용 뷰 |
| **프론트데스크 > 출발 화면** | "오늘 체크아웃 예정"만 필터링한 전용 뷰 |
| **Room Rack** | 완전 신규. 층/호수별 그리드 시각화 |
| **Walk-In** | 예약 생성 + 체크인을 한 프로세스로. 기존 예약 등록(reservation-form)과 별도 |

#### 4.3.4 예약관리 ↔ 프론트데스크 역할 분리 (권장)

```
[예약관리 메뉴] — 예약의 생애주기 관리
  ├── 예약 등록 (신규 예약 생성)
  ├── 예약 조회 (테이블뷰/캘린더뷰)
  ├── 예약 상세 (수정, 객실 추가/삭제, 결제, 상태 변경)
  └── 예약 채널 관리

[프론트데스크 메뉴] — 오늘의 실시간 운영
  ├── 대시보드 (도착/출발/투숙중 카운트 + KPI)
  ├── 도착 (오늘 체크인 대상 → 체크인 워크플로우)
  ├── 투숙 (현재 투숙중 → 객실 이동, 조기 체크아웃)
  ├── 출발 (오늘 체크아웃 대상 → 체크아웃 워크플로우)
  ├── Walk-In (현장 투숙 → 예약+체크인 동시)
  └── 객실현황 Room Rack (층/호수별 상태 그리드)
```

> **핵심 원칙**: 예약관리에서도 상태 변경(체크인/아웃)은 가능하지만,
> 프론트데스크는 **오늘의 운영에 최적화된 전용 뷰**를 제공.
> 동일한 백엔드 API를 공유하되, 프론트데스크에 **전제조건 검증 로직을 추가**.

## 5. 객실 상태 관리 (신규 필요)

### 5.1 현재 상태

- `htl_room_number` 테이블에 **객실 상태 필드 없음**
- 객실 가용성은 `SubReservation`의 체크인/체크아웃 날짜로만 관리
- 하우스키핑 상태, FO 상태 개념 자체가 없음

### 5.2 추가 필요 사항

#### DB 테이블 변경 (Flyway 마이그레이션)

```sql
-- htl_room_number 테이블에 객실 상태 필드 추가
ALTER TABLE htl_room_number ADD COLUMN hk_status VARCHAR(20) DEFAULT 'CLEAN';
ALTER TABLE htl_room_number ADD COLUMN fo_status VARCHAR(20) DEFAULT 'VACANT';
ALTER TABLE htl_room_number ADD COLUMN hk_updated_at TIMESTAMP;
ALTER TABLE htl_room_number ADD COLUMN hk_memo VARCHAR(500);
```

#### 상태 값 정의

**hk_status (하우스키핑 상태)** — Phase B에서 4종, Phase D에서 6종으로 확장:

| 값 | 한글 | Phase B | Phase D |
|----|------|:-------:|:-------:|
| `CLEAN` | 청소완료 | O | O |
| `DIRTY` | 청소필요 | O | O |
| `OOO` | 사용불가 (수리/공사) | O | O |
| `OOS` | 일시중단 | O | O |
| `INSPECTED` | 검사완료 | - | O |
| `PICKUP` | 간단정리 | - | O |

**fo_status (프론트오피스 상태)**:

| 값 | 한글 |
|----|------|
| `VACANT` | 비어있음 |
| `OCCUPIED` | 사용중 |

#### 자동 상태 전환 규칙

| 이벤트 | hk_status 변경 | fo_status 변경 |
|--------|---------------|---------------|
| 체크인 완료 | 변경 없음 (CLEAN 유지) | VACANT → **OCCUPIED** |
| 체크아웃 완료 | CLEAN → **DIRTY** | OCCUPIED → **VACANT** |
| 하우스키핑 청소 완료 | DIRTY → **CLEAN** | 변경 없음 |
| OOO 설정 (수동) | → **OOO** | → **VACANT** |
| OOS 설정 (수동) | → **OOS** | 변경 없음 |
| 야간감사 (Night Audit) | OC → **OD** (매일 밤) | 변경 없음 |

## 6. 구현 계획 (Phase별)

> 위 1~5번 내용 확정 후 순차 진행

### Phase A: 대시보드 고도화

**목표**: 역할별 대시보드 구현

| 작업 | 내용 |
|------|------|
| 대시보드 API | KPI 계산 서비스 (OCC%, ADR, RevPAR, 도착/출발/투숙 카운트) |
| PROPERTY_ADMIN 뷰 | 운영 대시보드 (도착/출발/투숙 + KPI + 객실 상태) |
| HOTEL_ADMIN 뷰 | 프로퍼티 비교 대시보드 |
| SUPER_ADMIN 뷰 | 포트폴리오 전체 대시보드 |
| 기존 dashboard.html | 확장 (정적 → 동적 데이터 바인딩) |

### Phase B: 프론트데스크 MVP

**목표**: 체크인/체크아웃 운영 화면

| 작업 | 내용 |
|------|------|
| DB 마이그레이션 | `hk_status`, `fo_status` 필드 추가 |
| RoomNumber 엔티티 | 상태 필드 추가 + 상태 전환 메서드 |
| changeStatus 확장 | 체크인 전제조건(객실배정) + 객실 상태 연동 |
| 도착(Arrivals) 화면 | 오늘 체크인 예정 리스트 + 체크인 워크플로우 (7단계) |
| 투숙(In-House) 화면 | 현재 투숙중 리스트 + 객실 이동/조기 체크아웃 |
| 출발(Departures) 화면 | 오늘 체크아웃 예정 리스트 + 체크아웃 프로세스 |
| Walk-In | 예약 생성 + 즉시 체크인 통합 프로세스 |

### Phase C: Room Rack

**목표**: 층/호수별 객실 상태 시각화

| 작업 | 내용 |
|------|------|
| Room Rack API | 프로퍼티 전체 객실 상태 조회 (층별 그룹핑) |
| Room Rack UI | 그리드 형태, 색상 코드 기반, 클릭 → 상세/상태 변경 |
| 상태 변경 UI | 프론트데스크에서 직접 Clean/Dirty/OOO/OOS 전환 |
| 실시간 갱신 | 주기적 폴링 또는 SSE 기반 상태 업데이트 |

### Phase D: 하우스키핑 연동

**목표**: 하우스키핑 전용 화면 + 프론트데스크 연동

| 작업 | 내용 |
|------|------|
| Task Sheet | 청소 담당자 배정, 객실 목록, 진행률 |
| INSPECTED/PICKUP | 상태 2종 추가 (관리자 검사, 간단 정리) |
| Room Discrepancy | FO↔HK 상태 불일치 감지/해결 |
| 모바일 대응 | 하우스키핑 담당자용 모바일 뷰 |

## 부록: 용어 사전 (Quick Reference)

| 영문 | 한글 | 설명 |
|------|------|------|
| OCC% | 점유율 | 전체 방 중 팔린 비율 |
| ADR | 평균객단가 | 팔린 방의 평균 가격 |
| RevPAR | 객실당수익 | 모든 방(빈 방 포함)의 평균 수익 |
| Arrivals | 도착 (체크인 예정) | 오늘 올 손님 |
| Departures | 출발 (체크아웃 예정) | 오늘 갈 손님 |
| In-House | 투숙중 | 지금 머물고 있는 손님 |
| Room Rack | 객실현황판 | 전체 방 상태를 한눈에 보는 판 |
| Walk-In | 현장투숙 | 예약 없이 와서 바로 투숙 |
| Day-Use | 당일사용 | 낮에만 쓰고 당일 체크아웃 |
| No-Show | 미도착 | 예약하고 안 온 경우 |
| Pickup | 예약증감 | 새 예약이 얼마나 늘었는지 |
| Vacant Clean (VC) | 빈 방·청소완료 | 바로 팔 수 있는 방 |
| Vacant Dirty (VD) | 빈 방·청소필요 | 청소해야 팔 수 있는 방 |
| Occupied Clean (OC) | 사용중·청소완료 | 손님 있고 오늘 청소 끝남 |
| Occupied Dirty (OD) | 사용중·청소필요 | 손님 있고 오늘 청소 안 함 |
| Out of Order (OOO) | 사용불가 | 수리/공사로 못 씀, 재고 제외 |
| Out of Service (OOS) | 일시중단 | 잠시 안 팔지만 급하면 가능 |
| Night Audit | 야간감사 | 하루 마감 처리 (매일 밤) |
| Folio | 투숙명세서 | 손님의 이용 내역서 |
| Queue | 대기 | 방이 아직 준비 안 된 경우 대기 |
| Task Sheet | 청소배정표 | 하우스키핑 담당자별 청소 목록 |
