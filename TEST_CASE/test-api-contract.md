# test-api-contract

## 목적
JWT 인증, 호텔/프로퍼티/층/호수/마켓코드 CRUD, 부킹엔진 API의 요청/응답 계약을 검증하기 위한 테스트 케이스 모음입니다.  
Claude Code의 `/test-api-contract` 스킬이 참고할 수 있도록 정상, 예외, 인증, 삭제 제약 시나리오를 정리합니다.

---

## API-001 JWT 로그인 성공
- **테스트케이스**: 올바른 관리자 계정으로 JWT 토큰을 정상 발급받아야 한다.
- **사전조건**:
  - 서버가 실행 중이어야 한다.
  - `admin / holapms1!` 계정이 존재해야 한다.
- **절차**:
  1. `POST /api/v1/auth/login` 요청을 전송한다.
  2. `loginId=admin`, `password=holapms1!` 값을 사용한다.
- **기대결과**:
  - HTTP 200 응답이어야 한다.
  - `success=true`가 반환되어야 한다.
  - `accessToken`, `refreshToken`, `userName`, `role` 필드가 포함되어야 한다.

## API-002 JWT 로그인 실패
- **테스트케이스**: 잘못된 비밀번호로 로그인 시 토큰 발급에 실패해야 한다.
- **사전조건**:
  - 서버가 실행 중이어야 한다.
- **절차**:
  1. `POST /api/v1/auth/login` 요청을 전송한다.
  2. 올바른 아이디와 잘못된 비밀번호를 사용한다.
- **기대결과**:
  - 인증 실패 응답이어야 한다.
  - 토큰이 발급되지 않아야 한다.
  - 에러 메시지 구조가 일관되어야 한다.

## API-003 호텔 등록 성공
- **테스트케이스**: 유효한 JWT로 호텔을 정상 등록할 수 있어야 한다.
- **사전조건**:
  - 유효한 JWT 토큰이 있어야 한다.
- **절차**:
  1. `POST /api/v1/hotels` 요청을 보낸다.
  2. `hotelCode`, `hotelName`, `hotelType`, `starRating`, `phone` 값을 포함한다.
- **기대결과**:
  - HTTP 200 또는 201이어야 한다.
  - 생성된 호텔 식별자 또는 생성 결과가 반환되어야 한다.
  - 저장 후 목록 조회 시 신규 호텔이 확인되어야 한다.

## API-004 호텔 등록 필수값 누락
- **테스트케이스**: 호텔 등록 시 필수값 누락 요청은 검증 오류로 차단되어야 한다.
- **사전조건**:
  - 유효한 JWT 토큰이 있어야 한다.
- **절차**:
  1. `POST /api/v1/hotels` 요청을 보낸다.
  2. `hotelCode` 또는 `hotelName`을 누락한다.
- **기대결과**:
  - HTTP 400이어야 한다.
  - 검증 실패 필드가 식별 가능해야 한다.
  - 데이터가 생성되지 않아야 한다.

## API-005 호텔 등록 인증 누락
- **테스트케이스**: 인증 없이 호텔 등록 요청 시 접근이 차단되어야 한다.
- **사전조건**:
  - 없음.
- **절차**:
  1. Authorization 헤더 없이 `POST /api/v1/hotels` 요청을 보낸다.
- **기대결과**:
  - HTTP 401 또는 403이어야 한다.
  - 인증 실패 응답 구조가 일관되어야 한다.
  - 호텔 데이터가 생성되지 않아야 한다.

## API-008 층 등록 성공

## API-009 호수 등록 - 중복 호수번호 검증
- **테스트케이스**: 동일 프로퍼티 내 중복 호수번호 등록 시 실패해야 한다.
- **사전조건**:
  - 프로퍼티에 호수 데이터가 존재해야 한다.
  - 유효한 JWT 토큰이 있어야 한다.
- **절차**:
  1. 이미 존재하는 `roomNumber`를 포함하여 `POST /api/v1/properties/{propertyId}/room-numbers` 요청을 보낸다.
- **기대결과**:
  - HTTP 409 또는 400이어야 한다.
  - 중복 관련 오류 메시지가 반환되어야 한다.
  - 호수 데이터가 생성되지 않아야 한다.
- **참고**: 층-호수 관계는 `RoomTypeFloor` 매핑 테이블(hola-room 모듈)로 관리되며, 호수 생성 API(`RoomNumberRequest`)에는 `floorId` 파라미터가 없다.
