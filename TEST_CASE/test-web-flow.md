# test-web-flow

## 목적
웹/모바일 웹 기준 핵심 사용자 흐름을 검증하기 위한 테스트 케이스 모음입니다.  
Claude Code의 `/test-web-flow` 스킬이 참고할 수 있도록 기능 흐름, 사전조건, 절차, 기대결과를 정리합니다.

---

## WF-001 관리자 로그인 성공
- **테스트케이스**: 관리자 계정으로 정상 로그인할 수 있어야 한다.
- **사전조건**:
  - 서버가 실행 중이어야 한다.
  - `admin / holapms1!` 계정이 존재해야 한다.
- **절차**:
  1. `/login` 페이지에 접속한다.
  2. 아이디 `admin`을 입력한다.
  3. 비밀번호 `holapms1!`를 입력한다.
  4. 로그인 버튼을 클릭한다.
- **기대결과**:
  - 로그인에 성공해야 한다.
  - `/admin/dashboard`로 이동해야 한다.
  - 인증 상태가 유지되어야 한다.

## WF-002 관리자 로그인 실패
- **테스트케이스**: 잘못된 비밀번호 입력 시 로그인에 실패해야 한다.
- **사전조건**:
  - 서버가 실행 중이어야 한다.
- **절차**:
  1. `/login` 페이지에 접속한다.
  2. 아이디 `admin`을 입력한다.
  3. 잘못된 비밀번호를 입력한다.
  4. 로그인 버튼을 클릭한다.
- **기대결과**:
  - 로그인에 실패해야 한다.
  - 에러 메시지가 표시되어야 한다.
  - 로그인 페이지에 그대로 머물러야 한다.

## WF-003 호텔 등록 성공
- **테스트케이스**: 관리자 화면에서 호텔을 정상 등록할 수 있어야 한다.
- **사전조건**:
  - SUPER_ADMIN 계정으로 로그인되어 있어야 한다.
  - `/admin/hotels` 페이지에 접근 가능해야 한다.
- **절차**:
  1. `/admin/hotels` 페이지로 이동한다.
  2. `호텔 등록` 버튼을 클릭한다.
  3. 호텔코드, 호텔명, 호텔유형, 등급, 전화번호를 입력한다.
  4. 저장 버튼을 클릭한다.
- **기대결과**:
  - 저장 성공 메시지가 표시되어야 한다.
  - 목록에 신규 호텔이 노출되어야 한다.
  - 중복 생성 없이 1건만 등록되어야 한다.

## WF-004 호텔 등록 필수값 검증
- **테스트케이스**: 호텔 등록 시 필수값이 누락되면 저장이 차단되어야 한다.
- **사전조건**:
  - SUPER_ADMIN 계정으로 로그인되어 있어야 한다.
- **절차**:
  1. 호텔 등록 모달을 연다.
  2. 필수값 일부를 비운 상태로 입력한다.
  3. 저장 버튼을 클릭한다.
- **기대결과**:
  - 저장이 차단되어야 한다.
  - 누락 필드에 대한 검증 메시지가 표시되어야 한다.
  - 데이터가 생성되지 않아야 한다.

## WF-005 프로퍼티 등록 성공
- **테스트케이스**: 호텔 상세 화면에서 프로퍼티를 정상 등록할 수 있어야 한다.
- **사전조건**:
  - 호텔 데이터가 1건 이상 존재해야 한다.
  - 관리자 로그인 상태여야 한다.
- **절차**:
  1. 호텔 목록에서 특정 호텔 상세로 진입한다.
  2. `프로퍼티 등록` 버튼을 클릭한다.
  3. 프로퍼티코드, 프로퍼티명, 유형, 체크인/체크아웃 시간, 총 객실수를 입력한다.
  4. 저장 버튼을 클릭한다.
- **기대결과**:
  - 프로퍼티가 정상 등록되어야 한다.
  - 해당 호텔 하위 목록에 신규 프로퍼티가 표시되어야 한다.
  - 호텔-프로퍼티 관계가 정상 유지되어야 한다.

## WF-006 프로퍼티 상세 탭 전환
- **테스트케이스**: 프로퍼티 상세 화면에서 층/호수/마켓코드 탭 전환이 정상 동작해야 한다.
- **사전조건**:
  - 프로퍼티 데이터가 존재해야 한다.
- **절차**:
  1. 프로퍼티 상세 페이지에 진입한다.
  2. `층 관리` 탭을 확인한다.
  3. `호수 관리` 탭을 클릭한다.
  4. `마켓코드` 탭을 클릭한다.
- **기대결과**:
  - 각 탭이 정상 전환되어야 한다.
  - 활성 탭 UI가 정확히 표시되어야 한다.
  - 탭 간 데이터가 섞이지 않아야 한다.

## WF-010 부킹엔진 예약 완료
- **테스트케이스**: 게스트가 객실 검색부터 예약 완료까지 정상 진행할 수 있어야 한다.
- **사전조건**:
  - 테스트용 프로퍼티와 가용 객실 데이터가 존재해야 한다.
  - `/booking/GMP` 접근이 가능해야 한다.
- **절차**:
  1. `/booking/GMP`에 접속한다.
  2. 체크인/체크아웃/인원수를 입력한다.
  3. `객실 검색`을 클릭한다.
  4. 객실 및 요금을 선택한다.
  5. `예약하기`를 클릭한다.
  6. 투숙객 정보를 입력하고 `예약 완료`를 클릭한다.
- **기대결과**:
  - 각 단계 페이지가 정상 이동해야 한다.
  - 예약 완료 페이지에서 확인번호가 발급되어야 한다.
  - PMS 예약 목록에 데이터가 적재되어야 한다.

## WF-011 하우스키퍼 모바일 로그인
- **테스트케이스**: 하우스키퍼 계정으로 모바일 로그인 후 권한에 맞는 화면으로 진입해야 한다.
- **사전조건**:
  - 하우스키퍼 계정(`hk_staff1 / holapms1!`)이 존재해야 한다.
  - `/m/housekeeping/login` 접근이 가능해야 한다.
- **절차**:
  1. `/m/housekeeping/login`에 접속한다.
  2. 아이디와 비밀번호를 입력한다.
  3. 로그인 버튼을 클릭한다.
- **기대결과**:
  - 로그인에 성공해야 한다.
  - 모바일 하우스키핑 첫 화면으로 이동해야 한다.
  - 관리자 화면으로 잘못 진입하지 않아야 한다.
