| 3-4 | DU-3H 적용 시 알림 | ✅ "Dayuse 레이트코드가 선택되었습니다. 체크아웃이 자동 설정됩니다." |

**실패 원인:**
- `reservation-form.js:1126`에서 subReservation에 `stayType`과 `dayUseDurationHours`를 전송하지 않음
- 서버가 `property.dayUseDefaultHours` (5시간)로 fallback → DU-3H에 5시간 요금 없음 → 에러
- **수정 필요**: JS에서 dayuse RC 선택 시 해당 RC의 durationHours를 자동 포함하거나, 이용시간 선택 UI 추가

**Flow #3 결과: ❌ FAIL (Dayuse 예약 UI 생성 불가)**
